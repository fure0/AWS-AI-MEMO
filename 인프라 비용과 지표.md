# MAU / DAU / RPU / Payload Size / Concurrency 개념과 인프라 비용 관계

## 1. MAU / DAU — 사용자 규모

### 개념

- **MAU (Monthly Active Users)**
  - 한 달 동안 서비스를 이용한 사용자 수

- **DAU (Daily Active Users)**
  - 하루 동안 서비스를 이용한 사용자 수

쉽게 말하면:

> "우리 서비스에 사람이 얼마나 오는가"

---

## 비용과의 관계

MAU / DAU 자체가 직접 비용을 만들지는 않지만,
모든 트래픽 계산의 시작점이 된다.

예:

- DAU 1만 명
- DAU 10만 명

→ 요청량, 서버 부하가 약 10배 차이

추가 지표:

```
DAU / MAU = 사용자 활성도
```

예:

```
DAU = 30,000
MAU = 100,000

DAU / MAU = 30%
```

---

# 2. RPU (Requests Per User)

## 개념

사용자 1명이 발생시키는 요청 수

예:

- SNS 서비스
  - 하루 100~1000 requests/user

- 단순 조회 서비스
  - 하루 10~50 requests/user

---

## 비용과의 관계

핵심 공식:

```
총 요청 수 = DAU × RPU
```

예:

```
DAU = 10,000
RPU = 100

10,000 × 100

= 1,000,000 requests/day
```

영향받는 비용:

- ALB 요청 처리 비용
- API 서버 CPU 사용량
- Lambda 호출 비용
- DB Query 수
- Cache 사용량

---

# 3. Payload Size (응답 크기)

## 개념

요청(Request) 또는 응답(Response)의 데이터 크기

예:

- 일반 REST API JSON
  - 1KB ~ 10KB

- 이미지 포함 API
  - 100KB ~ MB 단위

---

## 비용과의 관계

공식:

```
총 네트워크 트래픽
=
요청 수 × Payload Size
```

예:

```
요청 수 = 1,000,000/day
Payload = 10KB

1,000,000 × 10KB

= 10GB/day
```

영향받는 비용:

- CloudFront 비용
- AWS Data Transfer 비용
- Load Balancer 처리량
- 네트워크 비용

---

# 4. Concurrency (동시 접속자)

## 개념

특정 순간에 동시에 서비스를 이용하는 사용자 수

주의:

```
DAU ≠ Concurrency
```

예:

```
DAU = 100,000명

하지만 동시에 접속하는 사람은
1,000명일 수 있음
```

---

## 비용과의 관계

Concurrency는 서버 용량 결정 기준

영향:

- ECS Task 개수
- EC2 Instance 개수
- Lambda 동시 실행 수
- DB Connection Pool


예:

```
동시 사용자 100명
→ 서버 2대

동시 사용자 1000명
→ 서버 10대
```

---

# 전체 연결 구조

```
MAU
 ↓
DAU
 ↓
DAU × RPU
 ↓
총 요청량
 ↓
총 요청량 × Payload Size
 ↓
네트워크 비용 / 처리량

그리고

Concurrency
 ↓
필요한 서버 용량 결정
```

---

# AWS 비용 추정 예시

가정:

```
DAU = 50,000
RPU = 80
Payload Size = 5KB
Concurrency = 500
```

---

## 1) 하루 요청량

```
50,000 × 80

= 4,000,000 requests/day
```

---

## 2) 데이터 처리량

```
4,000,000 × 5KB

= 20GB/day

월 기준:

약 600GB/month
```

---

## 3) 서버 용량 결정

Concurrency:

```
500 concurrent users
```

예:

```
ECS Task 1개가 100명 처리 가능

500 / 100

= 최소 5 Task 필요
```

---

# 실무에서 사용하는 추정 방식

| 항목 | 추정 방법 |
|---|---|
| DAU | MAU의 10~30% |
| RPU | 서비스 특성 기반 |
| Payload Size | API 설계 기반 |
| Concurrency | DAU의 1~5% 수준 |

---

# 핵심 정리

## DAU × RPU

> 얼마나 많은 요청이 발생하는가

영향:
- API 서버
- DB
- ALB 비용


## Payload Size

> 데이터가 얼마나 큰가

영향:
- 네트워크 비용
- CDN 비용
- 데이터 전송 비용


## Concurrency

> 순간적으로 얼마나 몰리는가

영향:
- 서버 개수
- Auto Scaling 기준


최종적으로:

```
사용자 규모
(MAU / DAU)

+

사용 패턴
(RPU)

+

데이터 크기
(Payload Size)

+

순간 부하
(Concurrency)

=

필요한 인프라 규모와 비용
```
